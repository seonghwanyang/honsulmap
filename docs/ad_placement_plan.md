> 참고 자료: `ref/` 폴더 9장 이미지 (거지맵 광고 적용 사례 기준)
> 적용 범위: 맵 · 피드 · 커뮤니티 · 게시글 상세 · 스팟 상세 · 글쓰기
> 광고 상품: 네이티브 / 사이드 배너 / 플로팅 배너 / 하단 고정 바

## 1. Ref 이미지 광고 포맷 분해

### 1-1. 거지맵에서 관찰된 광고 슬롯

| # | 포맷 | 위치 | 사이즈 (추정) | 동작 | 근거 이미지 |
|---|---|---|---|---|---|
| A | 좌측 사이드 배너 | 본문 좌측 바깥 | 160×600 ~ 300×600 세로형 | sticky, 스크롤 따라감 | copy2, copy3 |
| B | 우측 사이드 배너 | 본문 우측 바깥 | 160×600 ~ 300×600 세로형 | sticky | image, copy2, copy3, copy4, copy6~8 |
| C | 플로팅 배너 | 우측 하단 fixed | 정사각/세로 소형 | × 닫기, 스크롤 무관 고정 | image, copy3, copy7 |
| D | 하단 가로 배너 | 뷰포트 bottom fixed | 전체 폭 × 60~90px | 항상 노출, 닫기 버튼 | copy2, copy6 |
| E | 사이드바 내부 네이티브 | 좌측 리스트/랭킹 맨 아래 | 리스트 아이템과 동일 | 섞여 있음 | image, copy |
| F | 리스트 인라인 네이티브 | 피드·게시판 N번째 | 카드와 동일 | "AD" 뱃지 외 UI 동일 | copy, copy2 |
| G | 상세 모달 하단 네이티브 | 스팟 팝업 내부 | 카드형 | 스팟 정보 직하단 | copy4, copy8 |
| H | 본문 중간 가로 네이티브 | 게시글 본문/댓글 사이 | 전체 폭 × 100~120px | 콘텐츠 흐름 차단 최소 | copy3 |

### 1-2. 핵심 원칙 (ref에서 추출)
- **광고 톤 일관성**: 네이티브는 기존 카드/리스트와 동일한 radius·border·padding 사용, 상단 "AD" 혹은 브랜드 네임만 노출
- **다중 슬롯 공존**: 맵 한 화면에 사이드(B) + 플로팅(C) + 하단바(D) 동시 노출 허용
- **×닫기 제공**: 플로팅/하단바에는 닫기 버튼 필수
- **모바일 다운그레이드**: 사이드 배너는 데스크톱 전용 → 모바일에선 하단 고정 바로 대체

## 6. 컴포넌트 설계 제안

### 6-1. 신규 컴포넌트
```
src/components/ads/
├── SideBanner.tsx       # A, B — position="left"|"right"
├── FloatingBanner.tsx   # C — dismissible, localStorage 기억
├── BottomStickyBar.tsx  # D — 모바일 전용, BottomNav 위
├── NativeCard.tsx       # E, G — 카드형 네이티브
├── NativeListItem.tsx   # F — 리스트 아이템형 네이티브
├── NativeHorizontal.tsx # H — 가로 전체폭 네이티브
└── AdSlot.tsx           # 공통 래퍼 (impression 추적, 광고 소스 추상화)
```

### 6-2. 공통 Props 설계
```ts
interface AdSlotProps {
  slotId: string;              // 트래킹용 고유 ID
  format: 'side' | 'floating' | 'bottom' | 'native-card' | 'native-list' | 'native-horizontal';
  breakpoint?: 'mobile' | 'desktop' | 'all';
  dismissible?: boolean;       // C, D용
  persistDismissalKey?: string; // localStorage 키
  testMode?: boolean;          // 광고 플레이스홀더 모드
}
```

### 6-3. 레이아웃 통합
- `app/layout.tsx`: 전역 `SideBanner` (좌/우) + `FloatingBanner` + `BottomStickyBar` 추가
- 페이지별 `metadata` 또는 route segment config로 슬롯 on/off 제어
- `/write`처럼 광고 억제 페이지는 `<AdSuppression />` 또는 context 플래그 사용

## 7. UX·정책 가이드

### 7-1. 노출 제약
- **동일 화면 광고 밀도 상한**: 뷰포트당 최대 3개 슬롯 동시 노출
- **네이티브 "AD" 뱃지 필수**: 좌상단 4px radius, 우리 Pretendard 10~11px
- **×닫기 후 재노출**: 세션 단위 숨김 (새로고침 시 리셋 또는 7일 localStorage)

### 7-2. 반응형 전환 규칙
| 폭 | 사이드 좌 | 사이드 우 | 플로팅 | 하단 바 |
|---|---|---|---|---|
| < 768 | — | — | — | ✅ |
| 768~1023 | — | 협소형 | ✅ 소형 | ✅ |
| 1024~1439 | — | 풀 | ✅ | — |
| ≥ 1440 | 풀 | 풀 | ✅ | — |

### 7-3. 성능
- 사이드/플로팅: lazy load, viewport intersection 기준 로딩
- 인라인 네이티브: 리스트 virtualization과 충돌 없게 `data-ad-slot` 마커 활용
- CLS 방지: 모든 슬롯은 지정 높이 사전 예약 (min-height)

### 7-4. 글로벌 스타일 톤
- 네이티브: 기존 spot/post 카드의 `rounded-xl`, `border border-neutral-200`, `bg-white` 그대로 차용
- 사이드/플로팅: 모노크롬 테두리 + 내부만 크리에이티브
