## 1. 프로젝트 요약

제주도 혼술바/게스트하우스의 인스타 스토리를 모아보고, 실시간 현황을 공유하는 모바일 퍼스트 웹사이트.

### 기술 스택
- 프론트: Next.js (App Router) + TypeScript
- DB/Storage: Supabase
- 배포: Vercel
- 지도: 네이버맵 API
- 인스타 수집: instaloader (Python) + GitHub Actions cron
- 광고: 카카오 AdFit

### 핵심 원칙
- 모바일 퍼스트 (PC 레이아웃은 MVP 이후)
- 회원가입 없음 — 익명 닉네임 + 비밀번호
- 87개 가게 시드 데이터로 시작

## 3. 데이터 모델

### spots (가게)
87개 혼술바/게하 정보. 시드 데이터로 미리 등록.

| 필드 | 설명 | 필수 |
|------|------|------|
| id | 고유 ID (UUID) | 자동 |
| name | 가게 이름 | O |
| slug | URL용 고유 문자열 (예: "ollebar-jeju") | O |
| instagram_id | 사장님 인스타 아이디 (스토리 수집용) | 선택 |
| category | 'bar' / 'guesthouse' / 'pub' / 'wine_bar' / 'karaoke_bar' | O |
| region | 'jeju' / 'aewol' / 'seogwipo' / 'east' / 'west' | O |
| address | 주소 | O |
| lat, lng | 위도/경도 (지도 핀 좌표) | O |
| phone | 전화번호 | 선택 |
| business_hours | 영업시간 | 선택 |
| memo | 메모/특이사항 | 선택 |
| naver_place_id | 네이버지도 장소 ID | 선택 |
| like_count | 찜 수 (하트) | 자동, 기본 0 |
| mood_up | 분위기 좋다 투표 수 | 자동, 기본 0 |
| mood_down | 분위기 별로 투표 수 | 자동, 기본 0 |
| image_urls | 가게 사진 URL 배열 | 선택 |
| created_at | 등록 시간 | 자동 |

> **찜 vs 분위기**: 찜(like_count)은 "이 가게 관심있어"고, 분위기(mood_up/down)는 "이 가게 분위기가 좋다/별로다" 평가. 별개 기능.

### stories (인스타 스토리 캐시)
GitHub Actions가 30분마다 수집해서 저장. 프론트는 여기서 읽기만.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| spot_id | 연결된 가게 |
| instagram_id | 인스타 계정 |
| media_url | 스토리 이미지/영상 URL (인스타 CDN) |
| media_type | 'image' / 'video' |
| thumbnail_url | 영상일 때 썸네일 |
| posted_at | 스토리 게시 시간 |
| expires_at | 만료 시간 (24시간 후) |
| scraped_at | 수집 시간 |

### posts (커뮤니티 게시글)
유저가 작성하는 현황/후기/꿀팁/자유 게시글.

| 필드 | 설명 | 필수 |
|------|------|------|
| id | 고유 ID | 자동 |
| spot_id | 연결된 가게 (현황/후기만) | 조건부 |
| category | 'status' / 'review' / 'tip' / 'free' | O |
| title | 제목 | O |
| content | 본문 | O |
| nickname | 작성자 닉네임 | O |
| password_hash | 비밀번호 해시 (수정/삭제용) | O |
| image_urls | 첨부 이미지 URL 배열 | 선택 |
| like_count | 찜 수 | 자동, 기본 0 |
| comment_count | 댓글 수 | 자동, 기본 0 |
| created_at | 작성 시간 | 자동 |

> **규칙**: 현황/후기 게시글은 반드시 가게(spot_id)를 선택해야 함. 꿀팁/자유는 가게 선택 없음.

### comments (댓글)
가게 댓글과 게시글 댓글 공용. 게시글 댓글만 대댓글 지원.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| post_id | 게시글 ID (게시글 댓글일 때) |
| spot_id | 가게 ID (가게 댓글일 때) |
| parent_id | 부모 댓글 ID (대댓글일 때) |
| nickname | 작성자 닉네임 |
| password_hash | 비밀번호 해시 |
| content | 댓글 내용 |
| like_count | 좋아요 수 |
| created_at | 작성 시간 |

> **규칙**: post_id 또는 spot_id 중 하나만 있어야 함 (둘 다 있거나 둘 다 없으면 안 됨). 가게 댓글에는 대댓글(parent_id) 불가.

### likes (찜/좋아요)
중복 방지용. 로그인 없으니 브라우저 핑거프린트로 구분.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| target_type | 'spot' / 'post' / 'comment' |
| target_id | 대상 ID |
| fingerprint | 브라우저 핑거프린트 |
| created_at | 시간 |

> **유니크 제약**: target_type + target_id + fingerprint 조합이 유일해야 함 → 한 사람이 같은 대상 중복 찜 불가.

### mood_votes (분위기 투표)
가게별 분위기 업다운. 마찬가지로 핑거프린트 기반 중복 방지.

| 필드 | 설명 |
|------|------|
| id | 고유 ID |
| spot_id | 가게 ID |
| vote | 'up' / 'down' |
| fingerprint | 브라우저 핑거프린트 |

> **유니크 제약**: spot_id + fingerprint 조합 유일. 같은 사람이 up 누른 상태에서 down 누르면 변경. 같은 버튼 다시 누르면 취소.

### contribution_rankings (기여랭킹)
최근 30일 기준 게시글 작성 수 + 받은 찜 수로 점수 계산. DB 뷰로 구현.

| 필드 | 설명 |
|------|------|
| nickname | 닉네임 |
| post_count | 게시글 수 |
| status_count | 현황 게시글 수 |
| review_count | 후기 게시글 수 |
| total_likes | 받은 총 찜 수 |
| score | 종합 점수 (게시글×10 + 찜수) |

> TOP 50만 표시.

## 4. API 설계

### 가게 (Spots)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/spots | 가게 목록. 필터: region, category |
| GET | /api/spots/:slug | 가게 상세 (스토리 + 댓글 포함) |
| POST | /api/spots | 가게 등록 (관리용) |
| PATCH | /api/spots/:id | 가게 수정 |
| POST | /api/spots/:id/like | 찜 토글. body: { fingerprint } |
| POST | /api/spots/:id/mood | 분위기 투표. body: { vote, fingerprint } |

### 스토리 (Stories)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/spots/:id/stories | 가게별 스토리 목록 (만료 제외) |
| GET | /api/stories/latest | 최신 스토리 전체 (피드 masonry용) |

### 게시글 (Posts)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/posts | 게시글 목록. 필터: category, spot_id |
| GET | /api/posts/:id | 게시글 상세 |
| POST | /api/posts | 게시글 작성 |
| PATCH | /api/posts/:id | 게시글 수정. body에 password 필수 |
| DELETE | /api/posts/:id | 게시글 삭제. body에 password 필수 |
| GET | /api/posts/popular | 인기글 (찜 기준, 캐러셀용) |

### 댓글 (Comments)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/comments?post_id=X | 게시글 댓글 (대댓글 트리 포함) |
| GET | /api/comments?spot_id=X | 가게 댓글 (flat) |
| POST | /api/comments | 댓글 작성 |
| DELETE | /api/comments/:id | 댓글 삭제. body에 password 필수 |
| POST | /api/comments/:id/like | 댓글 좋아요 토글 |

### 랭킹
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/rankings/contributions | 기여랭킹 TOP 50 |

## 5. 페이지별 상세 스펙

### 5.1 맵 페이지 (`/`)

**화면 구성 (위→아래)**:
1. 헤더: 서비스명 "제주혼술" + "현황 제보" 버튼
2. 지역 필터 칩: 전체 / 제주시 / 애월 / 서귀포 / 동쪽 / 서쪽
3. 지도 (풀스크린): 네이버맵, 가게별 핀
4. 맵 위 플로팅 띠배너 광고 (320×50)
5. FAB 2개: "목록" (바텀시트로 가게 리스트), "GPS" (내 위치)
6. FAB: 글쓰기 "+" (→ /write)
7. 하단 네비바: 맵 / 피드 / 커뮤
8. 하단 배너 광고 (네비 아래)

**핀 표시 규칙**:
- 핀 내용: `가게명` (스토리 있으면 `가게명 · 30분 전`)
- 핀 스타일: pill 형태 (border-radius 999px), 흰 배경
- 핀 테두리색: 24시간 내 수집된 스토리 있으면 초록, 없으면 회색
- 핀 클릭 → `/spot/:slug`로 이동

**지역 필터**:
- 칩 클릭 → 해당 지역 가게만 핀 표시
- "전체" 클릭 → 필터 해제
- URL에 `?region=aewol` 반영

### 5.2 피드 페이지 (`/feed`)

**화면 구성**:
1. 헤더: "제주혼술" + 검색 버튼
2. 지역 필터 칩 (맵과 공유)
3. masonry 2열 그리드: 가게별 카드
4. 카드 5개마다 네이티브 광고 (320×50)
5. 하단 네비바 + 하단 배너 광고

**카드 구성**:
- 최신 스토리 썸네일 (비율 유지, 3:4 또는 4:5)
- 가게명
- 지역 · 카테고리
- 배지: "스토리 N개" (24시간 내 활성 스토리 있을 때). 없으면 배지 없음
- 카드 클릭 → `/spot/:slug`로 이동

**정렬**: 최신 스토리가 있는 가게 먼저, 그 다음 최근 현황 게시글 있는 가게, 나머지는 이름순.

### 5.3 커뮤니티 페이지 (`/community`)

**화면 구성**:
1. 헤더: "커뮤니티" + 검색 버튼
2. 카테고리 필터 칩: 전체 / 현황 / 후기 / 꿀팁 / 자유
3. 인기글 캐러셀 (가로 스크롤, 찜 기준 상위 5개)
4. 기여랭킹 TOP 50 (접기/펼치기)
5. 게시글 리스트 (무한스크롤)
6. 게시글 5개마다 인라인 광고 (320×50)
7. FAB: 글쓰기 "+" (→ /write)
8. 하단 네비바 + 하단 배너 광고

**게시글 아이템 구성**:
- 제목
- 카테고리 배지 (현황/후기/꿀팁/자유)
- 작성자 닉네임 + 상대시간
- 찜 수 (♡) + 댓글 수 (💬)
- 이미지 있으면 이미지 아이콘

**인기글 캐러셀**: 최근 7일 기준 찜 수 높은 순 5개. 각 항목: 제목 + 찜수 + 댓글수.

**기여랭킹**: 최근 30일 기준. 닉네임 + 게시글 수 + 점수. 기본 접힌 상태, 펼치면 TOP 50.

### 5.4 가게 상세 (`/spot/:slug`)

**화면 구성 (위→아래)**:
1. 뒤로가기 + 가게명
2. 가게 사진 가로 스크롤 스트립
3. 메타 정보 그리드: 영업시간 / 최근 스토리 시간 / 최근 현황 게시글
4. 액션 버튼 4개: 찜(♡+숫자) / 분위기⬆ / 네이버지도(외부링크) / 인스타(외부링크)
5. 인스타 스토리 섹션: 세로 스크롤, 각 스토리 9:16 비율
6. 인라인 광고 (320×100)
7. 댓글 섹션: 닉네임+비번+입력+등록 / 댓글 리스트 (flat, 대댓글 없음)

**스토리 섹션 규칙**:
- 만료(24시간)된 스토리 제외
- 최신순 정렬
- 스토리 없으면 "아직 스토리가 없습니다" + 인스타 바로가기 링크
- 이미지는 인스타 CDN에서 직접 로딩 (다운로드하지 않음)

**댓글 규칙**:
- 닉네임 자동 생성 + 셔플 버튼
- 비밀번호 4자 이상 필수
- 대댓글 없음 (flat 구조)
- 각 댓글: 닉네임 + 내용 + 좋아요(♡) + 상대시간 + ··· 메뉴(삭제)

### 5.5 게시글 상세 (`/post/:id`)

**화면 구성**:
1. 뒤로가기 + "커뮤니티"
2. 게시글 헤더: 제목 + 카테고리 배지 + 작성자 + 시간
3. 본문
4. 액션 버튼: 찜(♡+숫자) / 공유 / 신고
5. 인라인 광고 (320×100)
6. 댓글 섹션: 닉네임+비번+입력+등록 / 댓글 리스트 (대댓글 O)

**댓글 규칙**:
- 가게 댓글과 동일 기본 구조
- **대댓글 지원**: 댓글 클릭 → 대댓글 입력 폼 열림, 들여쓰기 표시
- 정렬: 최신순

### 5.6 글쓰기 (`/write`)

**화면 구성**:
1. 뒤로가기 + "글쓰기"
2. 닉네임 (자동생성 + 셔플) + 비밀번호
3. 카테고리 선택: 현황 / 후기 / 꿀팁 / 자유
4. 가게 선택 드롭다운 (현황/후기일 때만 표시)
5. 제목 입력
6. 내용 입력
7. 사진 첨부 (선택, 최대 5장)
8. 등록 버튼

**조건부 필드 규칙**:
- 현황/후기 → 가게 선택 필수
- 꿀팁/자유 → 가게 선택 숨김
- 제목, 내용, 닉네임, 비밀번호 비어있으면 등록 버튼 비활성

### 닉네임 생성 규칙
패턴: `{형용사}{명사}{숫자3자리}`

형용사: 신나는, 조용한, 달달한, 부지런한, 엉뚱한, 고독한, 소박한, 든든한, 다정한, 느긋한

명사: 파인애플, 고래, 라면, 토마토, 고양이, 감자, 만두, 호랑이, 오징어, 찰떡

숫자: 100~999 랜덤

예시: 신나는파인애플421, 조용한고래783

### 공통 UI 패턴

**하단 네비바**: 3개 항목 (📍맵 / 📷피드 / 💬커뮤). 현재 페이지 Amber 색상, 나머지 회색. 고정 위치. 모든 페이지에 표시.

**하단 배너 광고**: 네비바 바로 아래 (네비 위가 아님). 전 페이지 공통.

**FAB 글쓰기 "+"**: 네비바 위 우측에 떠있는 Amber 원형 버튼. 맵, 커뮤니티 페이지에서만 표시. 피드에서는 숨김.

**뒤로가기**: 상세 페이지에서 좌상단 원형 ← 버튼.

**찜 버튼**: ♡ + 숫자. 누르면 채워진 하트 + 숫자 +1. 다시 누르면 취소.

**분위기 업다운**: ⬆/⬇ 두 버튼. 한쪽 누르면 활성, 반대쪽 누르면 전환, 같은 쪽 다시 누르면 취소.

**필터 칩**: 선택된 칩은 Amber 배경 + 흰 텍스트. 미선택은 흰 배경 + 회색 보더.

**상대시간 표시 규칙**: "방금" (1분 미만), "N분 전" (60분 미만), "N시간 전" (24시간 미만), "N일 전" (7일 미만), "2026.04.13" (7일 이상)

## 8. 인스타 스크래핑

### 아키텍처
GitHub Actions가 30분마다 Python 스크립트(instaloader) 실행 → 87곳 중 instagram_id 있는 가게만 스토리 URL 수집 → Supabase stories 테이블에 저장. Next.js 프론트는 Supabase에서 읽기만.

### 세부 사항
- 실행 주기: 30분마다, 24시간 풀타임
- 수집 내용: 스토리 URL + 미디어 타입 + 게시 시간 (다운로드 아님, URL만)
- 배치: 10곳씩, 배치 간 3초 대기 (rate limit 방지)
- 만료 처리: 24시간 지난 스토리 자동 삭제
- instagram_id 없는 가게는 스킵
- 세션: 서비스 전용 인스타 계정 1개, 세션 쿠키를 GitHub Secrets에 저장
- 예상 사용량: ~720분/월 (GitHub Actions 무료 2000분 한도 내)

## 9. 테스트 케이스

### 9.1 가게 (Spots)

**목록 조회**
- 전체 조회 → 87개 가게 반환, 각 항목에 id/name/slug/lat/lng 포함
- region=jeju 필터 → 제주시 가게만 반환
- region=aewol 필터 → 애월 가게만 반환
- 존재하지 않는 region → 빈 배열

**상세 조회**
- 유효한 slug → 가게 정보 + 스토리 + 댓글 반환
- 존재하지 않는 slug → 404

**찜 토글**
- 처음 찜 → like_count +1, likes 레코드 생성
- 이미 찜 상태에서 다시 → like_count -1, likes 레코드 삭제
- 같은 fingerprint 중복 요청 → 토글 (생성↔삭제 반복)
- 다른 fingerprint → 별도 집계

**분위기 투표**
- 처음 up 투표 → mood_up +1
- up 상태에서 down → mood_up -1, mood_down +1 (전환)
- up 상태에서 up 다시 → mood_up -1 (취소)
- 다른 fingerprint → 별도 집계

### 9.2 게시글 (Posts)

**작성**
- 유효한 데이터 → 201 생성, password는 해시 저장 (평문 저장 안 됨)
- 현황인데 spot_id 없음 → 400 에러
- 후기인데 spot_id 없음 → 400 에러
- 자유인데 spot_id 없음 → 201 정상
- 제목 빈값 → 400 에러
- 내용 빈값 → 400 에러
- 비밀번호 4자 미만 → 400 에러

**수정/삭제**
- 비밀번호 일치 → 200 성공
- 비밀번호 불일치 → 403 에러
- 존재하지 않는 post_id → 404

**목록 조회**
- category=status 필터 → 현황 게시글만 반환
- 정렬: 최신순 (created_at DESC)
- 무한스크롤: limit + offset 페이지네이션

**인기글**
- limit=5 → 찜 수 높은 순 5개

### 9.3 댓글 (Comments)

**게시글 댓글**
- 작성 → 201, post의 comment_count +1
- 대댓글 작성 (parent_id 포함) → 201, parent 아래 중첩
- 조회 → 트리 구조 반환 (댓글 + replies 배열)

**가게 댓글**
- 작성 → 201
- 대댓글 시도 (parent_id 포함) → 400 "가게 댓글에는 대댓글 불가"
- 조회 → flat 배열

**삭제**
- 비밀번호 일치 → 200, comment_count -1
- 비밀번호 불일치 → 403

**좋아요**
- 토글 동작 (찜과 동일)

### 9.4 UI 컴포넌트

**하단 네비바**
- 3개 항목 렌더링: 맵, 피드, 커뮤
- 현재 경로에 맞는 항목 active (Amber 색상)
- 클릭 → 해당 라우트로 이동

**맵 핀**
- 스토리 있는 가게: "올레바 제주 · 30분 전" 표시
- 스토리 없는 가게: "올레바 제주" 만 표시
- 24시간 내 스토리 있으면 초록 테두리, 없으면 회색
- 클릭 → /spot/:slug 이동

**피드 카드**
- 스토리 썸네일 표시
- 24시간 내 스토리 있으면 "스토리 N개" 배지
- 스토리 없으면 배지 없음
- 클릭 → /spot/:slug 이동

**닉네임 입력**
- 마운트 시 랜덤 닉네임 자동 생성 ("형용사+명사+숫자" 패턴)
- 셔플 버튼 → 새 랜덤 닉네임
- 비밀번호 4자 미만에서 등록 시도 → 에러 메시지

**글쓰기 폼**
- "현황" 선택 → 가게 선택 표시
- "자유" 선택 → 가게 선택 숨김
- 필수 필드 비어있으면 → 등록 버튼 비활성
- 등록 성공 → /community로 이동

**찜 버튼**
- 초기: 빈 하트 + 숫자
- 클릭 → 채워진 하트 + 숫자 +1
- 재클릭 → 빈 하트 + 숫자 -1

**분위기 업다운**
- 초기: ⬆/⬇ 둘 다 비활성
- ⬆ 클릭 → ⬆ 활성, mood_up +1
- ⬆ 활성 상태에서 ⬇ → ⬆ 비활성 + ⬇ 활성, mood_up -1 mood_down +1
- ⬆ 활성 상태에서 ⬆ → 비활성, mood_up -1 (취소)

**지역 필터 칩**
- 6개 렌더링: 전체, 제주시, 애월, 서귀포, 동쪽, 서쪽
- 클릭 → 해당 지역 필터 적용 + 칩 Amber 활성
- "전체" 클릭 → 필터 해제
- 페이지 이동해도 필터 상태 유지

**스토리 스크롤**
- 유효한 스토리 세로 나열 (9:16 비율)
- 만료된 스토리 제외
- 스토리 없으면 "아직 스토리가 없습니다" + 인스타 바로가기

**게시글 댓글 (대댓글)**
- 댓글 + 대댓글 트리 렌더링 (대댓글은 들여쓰기)
- 댓글 클릭 → 대댓글 입력 폼 열림

### 9.5 인스타 스크래핑

- instagram_id 있는 가게만 체크 (없는 가게 스킵)
- 스토리 URL + 메타데이터 수집 → Supabase에 저장
- 같은 media_url 중복 저장 안 됨
- 24시간 지난 스토리 삭제
- 10곳씩 배치, 배치 간 3초 대기

### 9.6 광고

- 커뮤니티 게시글 5번째 뒤에 인라인 광고 존재
- 피드 카드 5번째 뒤에 인라인 광고 존재
- 하단 배너가 네비바 아래(DOM 순서)에 위치
- 맵 페이지에서 플로팅 배너가 지도 위 오버레이
